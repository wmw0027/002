# 用户登录模块技术方案文档

## 1. 概述

本文档描述了用户登录模块的技术实现方案。该模块提供基于邮箱和密码的登录功能，包含前端表单校验、Mock 数据接口调用、登录成功跳转及失败提示等核心功能。

## 2. 技术架构

- **前端框架**: 原生 HTML + CSS + JavaScript
- **数据模拟**: 使用 Mock.js 或手动模拟后端接口
- **通信方式**: 前端通过 `fetch` 或 `XMLHttpRequest` 调用 Mock API

## 3. 模块设计

### 3.1 页面结构

- 登录页面独立，路径为 `/login.html`
- 包含邮箱输入框、密码输入框、登录按钮、错误提示区域

### 3.2 表单校验规则

| 字段 | 规则 | 提示信息 |
|------|------|----------|
| 邮箱 | 必填，格式为邮箱格式 | 请输入有效邮箱地址 |
| 密码 | 必填，长度 6-20 位 | 密码长度需为 6-20 位 |

### 3.3 Mock 接口设计

- **接口地址**: `/api/login`
- **请求方式**: POST
- **请求参数**:
  ```json
  {
    "email": "user@example.com",
    "password": "123456"
  }
  ```
- **成功响应**:
  ```json
  {
    "code": 200,
    "message": "登录成功",
    "data": {
      "token": "mock_token_abc123",
      "user": {
        "name": "张三",
        "email": "user@example.com"
      }
    }
  }
  ```
- **失败响应**:
  ```json
  {
    "code": 401,
    "message": "邮箱或密码错误"
  }
  ```

### 3.4 交互流程

1. 用户输入邮箱和密码
2. 点击登录按钮，触发前端校验
3. 校验通过后，调用 Mock 登录接口
4. 接口返回成功 → 存储 token 到 localStorage → 跳转首页
5. 接口返回失败 → 显示错误提示信息

## 4. 任务

- **创建登录页面结构**: 使用 HTML 构建登录表单，包含邮箱输入框、密码输入框、登录按钮和错误提示区域。
- **实现表单前端校验**: 编写 JavaScript 校验函数，对邮箱格式和密码长度进行验证，校验失败时显示对应错误信息。
- **搭建 Mock 数据服务**: 使用 Mock.js 或手动拦截 fetch 请求，模拟后端登录接口，返回成功或失败响应。
- **实现登录接口调用**: 使用 fetch 发送 POST 请求到 `/api/login`，传递邮箱和密码参数。
- **处理登录成功逻辑**: 接口返回成功时，将 token 和用户信息存储到 localStorage，并跳转到首页。
- **处理登录失败逻辑**: 接口返回失败时，在页面上显示后端返回的错误提示信息。
- **优化用户体验**: 添加加载状态（按钮禁用、显示加载动画），防止重复提交。

## 5. 文件结构

```
/login
├── login.html          # 登录页面
├── css/
│   └── login.css       # 登录页面样式
├── js/
│   ├── login.js        # 登录逻辑（校验、调用接口、跳转）
│   └── mock.js         # Mock 数据服务
└── index.html          # 首页（登录成功后跳转）
```

## 6. 注意事项

- 前端校验仅用于提升用户体验，不可替代后端安全校验
- Mock 数据仅在开发环境使用，上线前需替换为真实接口
- token 存储建议使用 `sessionStorage` 或 `localStorage`，根据安全需求选择
- 密码传输建议使用 HTTPS 加密，Mock 环境下可忽略

# 用户登录模块技术方案文档

## 1. 概述

本文档描述用户登录模块的技术实现方案。该模块实现基于邮箱和密码的登录功能，包括前端表单展示、表单校验、与Mock API交互、登录成功/失败后的页面处理。

## 2. 技术架构

- **前端框架**: 纯 HTML + CSS + JavaScript（无第三方库）
- **数据模拟**: 使用 `mock.js` 或纯 JavaScript 模拟后端接口
- **API 调用**: 使用 `fetch` 或 `XMLHttpRequest` 进行接口请求
- **页面路由**: 单页应用或简单跳转（登录成功后跳转至 `/index.html`）

## 3. 组件设计

### 3.1 登录页面结构

```
login/
├── index.html          # 登录页面
├── css/
│   └── main.css        # 页面样式
├── js/
│   ├── main.js         # 入口逻辑
│   ├── validation.js   # 表单校验
│   ├── api.js          # API 封装
│   └── mock.js         # Mock 数据配置
```

### 3.2 登录表单组件

- **邮箱输入框**: `type="email"`，带 placeholder
- **密码输入框**: `type="password"`，带 placeholder
- **提交按钮**: 带加载状态
- **错误提示区域**: 用于显示校验或接口返回的错误信息

## 4. 功能实现

### 4.1 表单前端校验

- 邮箱格式校验：必须包含 `@` 和 `.`
- 密码非空校验：长度不小于6位
- 实时校验：输入时触发（可选）
- 提交时校验：阻止表单提交，显示错误提示

### 4.2 Mock 数据

- 预设固定用户数据：`email: user@example.com`，`password: 123456`
- 模拟登录接口响应延迟 `500ms~1000ms`
- 响应格式：
  - 成功: `{ code: 200, message: '登录成功', token: 'xxx' }`
  - 失败: `{ code: 401, message: '邮箱或密码错误' }`

### 4.3 登录流程

1. 用户填写邮箱和密码
2. 点击登录按钮
3. 前端执行表单校验
4. 校验通过 → 显示加载状态 → 调用登录API
5. API 返回成功 → 跳转至首页 (`/index.html`)
6. API 返回失败 → 显示错误提示信息

## 5. 错误处理

- 网络异常：提示“网络异常，请稍后重试”
- 接口返回错误：显示后端返回的错误消息
- 表单校验失败：显示具体的字段错误提示
- 防止重复提交：提交后禁用按钮，直到请求完成

## 6. 性能与安全

- 使用 `fetch` 的 `AbortController` 取消未完成的请求（可选）
- 不存储明文密码在本地
- 登录成功后存储 token 到 `localStorage`
- 错误提示避免泄露敏感信息

## 7. 任务

- **创建项目目录结构**: 创建 `login/` 目录并初始化 `index.html`、`css/main.css`、`js/` 子目录及文件。
- **实现登录表单页面**: 编写 HTML 结构，包含邮箱输入框、密码输入框、提交按钮及错误提示区域，并应用基础 CSS 样式。
- **实现表单前端校验**: 在 `validation.js` 中编写邮箱格式和密码长度的校验逻辑，并在提交时触发。
- **实现 Mock 数据与登录 API**: 在 `mock.js` 中模拟固定用户数据，在 `api.js` 中封装 `login(email, password)` 方法，返回 Promise。
- **实现登录逻辑与交互**: 在 `main.js` 中监听表单提交事件，执行校验后调用 API，处理成功（跳转首页）和失败（显示错误提示）逻辑。
- **处理加载状态与防重复提交**: 在 API 调用期间禁用提交按钮并显示加载动画，请求完成后恢复按钮状态。
- **集成测试与边界处理**: 测试正确/错误邮箱密码组合、空输入、网络异常等场景，确保错误提示正确显示。
